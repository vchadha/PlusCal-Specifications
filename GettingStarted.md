# TLA+ / PlusCal: Installation & Getting Started Guide

TLA+ is a formal specification language for describing and verifying the correctness of systems —
particularly concurrent and distributed ones. **PlusCal** is a higher-level algorithmic language
that compiles down to TLA+. Most people write PlusCal and let the tools translate it; you only
need to read raw TLA+ when debugging or writing advanced properties.

The toolchain revolves around **TLC**, the model checker. TLC exhaustively explores every possible
state your system can reach and checks that your invariants hold in all of them. When it finds a
violation it prints an exact step-by-step error trace showing you how you got there.

> **Prerequisite for all three sections:** Java 21+. See
> [Installing Java](#0-prerequisite-java-21-via-sdkman) below before continuing.

## 0. Prerequisite: Java 21 via SDKMAN

All TLA+ tooling runs on the JVM. The recommended way to install Java on macOS and Linux is
[SDKMAN](https://sdkman.io), which lets you manage multiple Java versions without touching system
paths.

### Install SDKMAN

```bash
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
```

### Install OpenJDK 21

```bash
sdk install java 21.0.3-ms   # Microsoft OpenJDK 21 — or pick any 21.x distribution
java -version                 # should print: openjdk version "21.x.x" ...
```

> **Tip:** Run `sdk list java` to see all available distributions and versions. Any `21.x` or
> later build works fine with TLA+.

**Windows:** SDKMAN works inside WSL. If you prefer a native install, download OpenJDK 21
directly from the [Microsoft OpenJDK page](https://learn.microsoft.com/en-us/java/openjdk/download#openjdk-21).

## The Toy Example: `Counter.tla`

Throughout this guide we use a single simple specification: a counter that starts at `0` and
increments by `1` on each step. Our invariant requires that it never exceeds `5`.

### `Counter.tla`

```tla
---- MODULE Counter ----
EXTENDS Integers

(*--algorithm Counter
variables
  count = 0;

begin
  Loop:
    while TRUE do
      count := count + 1;
    end while;
end algorithm; *)

\* BEGIN TRANSLATION - the following section is auto-generated by the PlusCal translator
\* END TRANSLATION

CounterInvariant == count <= 5

====
```

A few things to notice:

- The module name (`Counter`) **must match the filename** (`Counter.tla`), case-sensitively.
- The module is delimited by `---- MODULE Counter ----` at the top and `====` at the bottom.
- The PlusCal algorithm lives inside a `(*-- ... *)` block comment, so the file is valid TLA+
  even before translation.
- `CounterInvariant` is a TLA+ operator defined *outside* the algorithm. The model checker will
  verify it holds in every reachable state.
- The `\* BEGIN TRANSLATION` / `\* END TRANSLATION` markers are where the PlusCal translator
  injects the generated TLA+. **Do not edit between these markers manually.**

### `Counter.cfg`

The model checker needs a configuration file telling it what to check. Create `Counter.cfg` in
the same directory as `Counter.tla`:

```
SPECIFICATION Spec

INVARIANT CounterInvariant
```

- `SPECIFICATION Spec` tells TLC which operator defines the system's behavior. The PlusCal
  translator generates an operator called `Spec` (inside the `BEGIN TRANSLATION` block) that
  combines the initial state and the transition relation. This is what TLC uses to explore states.
- `INVARIANT CounterInvariant` tells TLC to verify our operator holds in every state it reaches.

> **Expected result:** TLC will find an invariant violation and stop. The error trace will show
> `count` stepping from `0` → `1` → … → `6`, at which point `CounterInvariant` (`count <= 5`)
> is false.

## Part 1 — VSCode Extension (Recommended)

The [official TLA+ Foundation VSCode extension](https://marketplace.visualstudio.com/items?itemName=tlaplus.vscode-ide)
(`tlaplus.vscode-ide`) is the fastest path to a productive TLA+ environment. It wraps the same
`tla2tools.jar` used by the Toolbox and CLI, but surfaces everything through VS Code's UI and
Command Palette.

### 1.1 Install the Extension

1. Open VS Code.
2. Go to the **Extensions** panel (`Ctrl+Shift+X` / `Cmd+Shift+X`).
3. Search for **TLA+** and install the one published by **TLA+ Foundation** (identifier:
   `tlaplus.vscode-ide`).

> **Note:** There is an older community extension (`alygin.vscode-tlaplus`) that you may
> encounter in older tutorials. The Foundation extension (`tlaplus.vscode-ide`) is the actively
> maintained successor — use that one.

Alternatively, open Quick Open (`Ctrl+P`) and run:
```
ext install tlaplus.vscode-ide
```

### 1.2 Workflow: Translate, Configure, Check

**Step 1 — Create your files.**

Create `Counter.tla` and `Counter.cfg` in the same directory using the content from the
[toy example](#the-toy-example-countertla) above.

**Step 2 — Parse and translate PlusCal.**

Open `Counter.tla` in VS Code and open the Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`).
Run:

```
TLA+: Parse module
```

This does two things in one step: it runs the PlusCal translator (filling in the
`BEGIN TRANSLATION` block) and then runs the SANY parser to check TLA+ syntax. Errors appear
in VS Code's **Problems** panel. On success, the translated TLA+ is written directly into
`Counter.tla`.

**Step 3 — Run the model checker.**

With `Counter.tla` open, run from the Command Palette:

```
TLA+: Check model
```

The extension automatically looks for `Counter.cfg` (same name, same directory). TLC output
streams into a dedicated panel. You will see an invariant violation — click the error in the
output panel to expand the full trace and step through each state.

> To run against a different `.cfg` file, use **`TLA+: Check model with non-default config...`**
> and select your file from the dialog.

### 1.3 Key Commands Reference

| Command | What it does |
|---|---|
| `TLA+: Parse module` | Translate PlusCal → TLA+, then check syntax with SANY |
| `TLA+: Check model` | Run TLC with the default `.cfg` file |
| `TLA+: Check model with non-default config...` | Run TLC with a chosen `.cfg` file |
| `TLA+: Run last model check again` | Re-run TLC without switching files |
| `TLA+: Stop model checking` | Kill the running TLC process |
| `TLA+: Evaluate constant expression` | Evaluate a selected TLA+ expression |
| `TLA+: Visualize TLC output` | Open the graphical results view |
| `TLA+: Export module to LaTeX` | Generate `.tex` / `.dvi` |
| `TLA+: Export module to PDF` | Generate a PDF document |

> **Keyboard shortcuts:** The extension ships with no default bindings to avoid conflicts with
> your existing setup. Open the Keyboard Shortcuts editor (`Ctrl+K Ctrl+S`), type `TLA+` to
> filter all TLA+ commands, and bind whichever you use most often.

### 1.4 Improving Your VSCode Experience

#### Parse on Save

Triggering `TLA+: Parse module` manually on every edit gets tedious quickly. You can make it
run automatically every time you save a `.tla` file. Open your `settings.json`
(`Ctrl+Shift+P` → *Preferences: Open User Settings (JSON)*) and add:

```json
"[tlaplus]": {
    "editor.codeActionsOnSave": {
        "source": true
    }
}
```

From now on, saving any `.tla` file automatically translates PlusCal and parses the module,
with errors surfacing in the Problems panel immediately.

#### Evaluate Constant Expressions

You can evaluate any TLA+ constant expression without running a full model check. Select the
expression in the editor and run `TLA+: Evaluate constant expression` from the Command Palette.
The result appears in the output panel.

This is handy for testing individual operators as you write them — for example, selecting
`{1, 2, 3} \intersect {2, 3, 4}` and evaluating it to confirm it returns `{2, 3}`.

#### State-Space Graph (dot / Graphviz)

TLC can export its explored state space as a Graphviz `.dot` file, letting you visually inspect
the states and transitions of small specs. Add the following to the **`TLC Model Checker: Options`**
setting (VS Code Settings → Extensions → TLA+):

```
-dump dot,colorize,actionlabels ${specName}.dot
```

The `${specName}` variable is automatically substituted with your spec's filename (without
extension), so running TLC on `Counter.tla` produces `Counter.dot`. The `colorize` option
colours edges by action and `actionlabels` annotates each edge with the action name.

Render the graph once generated:

```bash
dot -Tsvg Counter.dot -o Counter.svg
dot -Tpng Counter.dot -o Counter.png
```

> **Note:** TLC does not append a file extension automatically — always include `.dot` in the
> filename (or use `${specName}.dot` as shown above).

#### Export to LaTeX / PDF

- **`TLA+: Export module to LaTeX`** — generates `.tex` and `.dvi` files from the current spec.
- **`TLA+: Export module to PDF`** — generates a PDF document.

Both require a LaTeX distribution with `pdflatex` on your `PATH`:
- macOS: `brew install --cask mactex`
- Ubuntu/Debian: `sudo apt install texlive-full`

Additional PDF options are available under VS Code Settings → Extensions → TLA+: you can enable
line numbers (`PDF: Number Lines`), comment shading (`PDF: Comments Shade`), and choose the
shade colour (`PDF: Comments Shade Color`).

#### Boxed Comments

TLA+ specifications commonly use a distinctive boxed comment style:

```tla
(**************************************************************)
(* This is a boxed comment.                                   *)
(* Such comment format is used in many specifications.        *)
(**************************************************************)
```

VS Code does not format these natively, but the
[Comment Box extension](https://marketplace.visualstudio.com/items?itemName=slysherz.comment-box)
(`slysherz.comment-box`) adds this capability. After installing it, add the following to your
`settings.json` to configure the TLA+ style:

```json
"commentBox.styles": {
    "defaultStyle": {
        "commentStartToken": "(**",
        "commentEndToken": "**)",
        "leftEdgeToken": "(* ",
        "rightEdgeToken": " *)",
        "topEdgeToken": "*",
        "bottomEdgeToken": "*",
        "topRightToken": "**)",
        "bottomLeftToken": "(**",
        "capitalize": false,
        "ignoreInnerIndentation": false,
        "ignoreOuterIndentation": false,
        "removeEmptyLines": false,
        "textAlignment": "left"
    }
}
```

Select any comment text and invoke the Comment Box command to wrap it. Refer to the
[Comment Box documentation](https://github.com/slysherz/vscode-comment-box/blob/master/README.md)
for instructions on setting up a keyboard shortcut.

## Part 2 — TLA+ Toolbox (Legacy IDE)

The TLA+ Toolbox is the original dedicated IDE for TLA+, built on Eclipse. It is no longer
actively developed, but remains fully functional and well-documented. If you prefer a
self-contained GUI without VS Code, or need to follow older tutorials, the Toolbox works well.

### 2.1 Install the Toolbox

Download the latest release for your platform from the
[TLA+ GitHub releases page](https://github.com/tlaplus/tlaplus/releases/tag/v1.8.0). Java must
be installed and on your `PATH` (see [Section 0](#0-prerequisite-java-21-via-sdkman)).

Unzip the archive and run the `toolbox` binary (macOS/Linux) or `toolbox.exe` (Windows).

### 2.2 Configure External Tool Paths

Before using dot or PDF export, tell the Toolbox where to find those tools. Go to
**File → Preferences → TLA+ Preferences** and set:

- **Specify dot command** — the full path to your `dot` binary (e.g. `/usr/local/bin/dot`).
  Install Graphviz with `brew install graphviz` (macOS) or `sudo apt install graphviz` (Linux).
- **Specify pdflatex command** — the full path to your `pdflatex` binary (e.g.
  `/Library/TeX/texbin/pdflatex`). Install a LaTeX distribution with `brew install --cask mactex`
  (macOS) or `sudo apt install texlive-full` (Linux).

### 2.3 Create a Specification

1. Go to **File → Open Spec → Add New Spec**.
2. Browse to the directory where you saved `Counter.tla` and select it (or type a new path to
   create a new file).
3. The editor opens. Replace the blank skeleton with the contents of `Counter.tla` from the
   [toy example](#the-toy-example-countertla) above.

The module name in the first line must match the filename exactly.

### 2.4 Translate PlusCal

Go to **File → Translate PlusCal Algorithm**, or press `Ctrl+T` (Windows/Linux) / `Cmd+T`
(macOS). The Toolbox runs the translator and injects the generated TLA+ between the
`BEGIN TRANSLATION` / `END TRANSLATION` markers in your file.

### 2.5 Create a Model and Set Invariants

1. Go to **TLC Model Checker → New Model** and give it a name (e.g. `Model_1`).
2. In the model editor, confirm **"What is the behavior spec?"** is set to **Temporal Formula**
   with the value `Spec`. If it shows something different, check that the PlusCal translation
   ran and that there is only one `====` at the bottom of the file.
3. Scroll to the **Invariants** section and click **Add**. Type `CounterInvariant`.
4. Press **F11** (or the green Run button) to start TLC.

### 2.6 Read the Error Trace

When TLC finds the invariant violation, a red error panel appears on the right. This is the
**error trace** — an ordered list of states, each showing the value of every variable at that
step. You will see `count` increasing from `0` up through `6`, with the final state flagged as
the violating one.

Click any state row to expand it and inspect all variable values. The **Trace Explorer** panel
lets you add derived expressions (e.g. `count + 1`) to the trace output for richer debugging.

### 2.7 Generate a State-Space Graph

1. Open the model editor and go to the **TLC Options** tab.
2. Under **Features**, enable **"Visualize state graph after completion of model checking"**.
3. Run the model. When TLC finishes, the Toolbox renders the state graph using the `dot` binary
   configured in [Section 2.2](#22-configure-external-tool-paths).

### 2.8 Export to PDF

With a spec open, go to **File → Produce PDF Version of Module**. This invokes Tla2Tex using
the `pdflatex` path configured in [Section 2.2](#22-configure-external-tool-paths) and opens
the resulting PDF.

### 2.9 The Scratch File Pattern

When exploring TLA+ operators it helps to have a throwaway spec for quick evaluation. Create
`scratch.tla`:

```tla
---- MODULE scratch ----
EXTENDS Integers, TLC, Sequences

Eval == {1, 2, 3} \intersect {2, 3, 4}
====
```

Create a model for it. In the model editor set **"What is the behavior spec?"** to
**No behavior spec**, then on the **Model Checking Results** tab enter `Eval` in the
**Evaluate Constant Expression** box. Running the model prints the value of `Eval`. Change
the expression and re-run to explore any operator interactively.

## Part 3 — CLI (`tla2tools.jar`)

The CLI is ideal for scripting, CI pipelines, and automation. Everything the Toolbox and VSCode
extension do under the hood ultimately calls `tla2tools.jar`.

### 3.1 Get `tla2tools.jar`

Download the latest release from the
[TLA+ GitHub releases page](https://github.com/tlaplus/tlaplus/releases). Look for
`tla2tools.jar` in the release assets. If you have the Toolbox installed, `tla2tools.jar` also
lives in its base installation directory.

Place the jar somewhere convenient — either on your `PATH` or alongside your spec files.

### 3.2 Translate PlusCal

```bash
java -cp tla2tools.jar pcal.trans Counter.tla
```

This translates the PlusCal inside `Counter.tla` and writes the result back into the same file
between the `BEGIN TRANSLATION` / `END TRANSLATION` markers. It also:

- Creates `Counter.old` as a backup of the pre-translation file.
- Creates `Counter.cfg` if one does not already exist.

To prevent creating or overwriting `Counter.cfg`, add `-nocfg`:

```bash
java -cp tla2tools.jar pcal.trans -nocfg Counter.tla
```

There is no flag to suppress `Counter.old`. Add `find . -name "*.old" -delete` to your build
script if those files are unwanted.

### 3.3 Run TLC

```bash
java -jar tla2tools.jar -config Counter.cfg -workers auto Counter.tla
```

- `-config Counter.cfg` — points TLC at the configuration file. If omitted, TLC looks for a
  file named `<specname>.cfg` in the same directory automatically.
- `-workers auto` — use one worker thread per CPU core. **This matters significantly.** Without
  it, TLC defaults to a single thread and is much slower on larger models.

You should see an invariant violation with a trace showing `count` incrementing past `5`.

### 3.4 Generate a State-Space Graph (dot)

```bash
java -jar tla2tools.jar \
  -config Counter.cfg \
  -workers auto \
  -dump dot,colorize,actionlabels Counter.dot \
  Counter.tla
```

Render the graph:

```bash
dot -Tsvg Counter.dot -o Counter.svg
dot -Tpng Counter.dot -o Counter.png
```

> **Note:** TLC does not append a file extension automatically — always include `.dot` in the
> filename you pass.

### 3.5 Export to LaTeX / PDF

```bash
# Generate .tex and .dvi
java -cp tla2tools.jar tla2tex.TLA Counter.tla

# Generate PDF (requires pdflatex on PATH)
java -cp tla2tools.jar tla2tex.TLA -shade -number Counter.tla
```

`-shade` enables background shading on comments and the PlusCal block. `-number` adds line
numbers. See [Section 2.2](#22-configure-external-tool-paths) for `pdflatex` install
instructions.

### 3.6 Key TLC Flags

| Flag | Description |
|---|---|
| `-workers auto` | Use one thread per CPU core (strongly recommended) |
| `-config file.cfg` | Specify the config file (defaults to `<spec>.cfg` if omitted) |
| `-continue` | Keep checking after the first violation; reports all violations found |
| `-dump dot file` | Export state graph; no extension is appended automatically |
| `-dump dot,colorize,actionlabels file` | State graph with coloured, labelled edges |
| `-metadir dir` | Write TLC's state-space cache to `dir` instead of alongside the spec |
| `-fpmem 0.5` | Fraction of system RAM to allocate for fingerprint storage (default: `0.25`) |
| `-noGenerateSpecTE` | Suppress writing the error-trace spec file on violation |
| `-help` | Print all available flags (note: `-help`, not `-h`) |
